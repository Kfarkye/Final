Recommended MCP key
STRIPE_MCP_RESTRICTED_KEY
Build/test keys
Use test-mode keys such as rk_test_* or sk_test_*.
Idempotency
Use idempotency keys for mutating POST requests.
Webhook verification
Verify webhooks with the raw request body.
Logging policy
Never log secret keys, client_secret, payment_method, card data, bank account data, or raw customer PII.

Implementation Notes

Auth boundary
Truth backend should derive user identity from verified Firebase auth tokens, not from spoofable client headers such as x-user-id.
Artifact storage
Use Firebase Storage or Google Cloud Storage for uploaded files, generated artifacts, snapshots, and workspace-adjacent assets.
Security rules
Enforce per-user or per-workspace access at the rules layer and again in the backend service layer.
Metadata
Store canonical artifact metadata in Spanner and human-readable artifacts in Drive or Storage.

Implementation Notes

Current service
reverie on Cloud Run.
Region
us-central1
Deploy command pattern
gcloud run deploy reverie --source . --region us-central1
Secrets
Prefer Secret Manager bindings for Stripe, Linear, GitHub, and Firebase credentials instead of plain environment variables.
Health checks
Maintain /readyz for app readiness and add /api/mcp/status for MCP integration readiness.
MCP server packaging
Copy compiled MCP servers into /opt/truth/mcp-servers inside the container image.
